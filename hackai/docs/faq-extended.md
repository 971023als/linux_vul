## 기술적 세부 질문

### Q: RAG 파이프라인에서 간접 인젝션을 어떻게 완화하나요?

간접 인젝션 완화는 여러 계층의 방어가 필요합니다:

1. **입력 출처 추적**: 벡터 DB에 저장할 때 콘텐츠의 출처 URL과 신뢰 등급을 함께 저장합니다.
2. **콘텐츠 격리**: 검색된 외부 문서를 시스템 프롬프트와 명확히 구분되는 컨텍스트 영역에 배치합니다.
3. **출력 검증**: 에이전트가 검색 결과를 처리한 후의 출력을 구조화된 스키마로 파싱하고 검증합니다.
4. **정기 감사**: 벡터 DB 콘텐츠를 정기적으로 검토하여 오염된 항목을 제거합니다.

### Q: 멀티 에이전트 시스템에서 인증은 어떻게 구현하나요?

에이전트 간 통신에 서명된 메시지 또는 단기 토큰을 사용하여 발신 에이전트를 검증하는 것이 기본 원칙입니다. 구체적인 구현 방법은 사용하는 오케스트레이션 프레임워크와 인프라 환경에 따라 다릅니다. 각 프레임워크의 공식 보안 문서를 참조하세요.

### Q: 에이전트 행동 기준선은 어떻게 정의하나요?

초기 운영 기간 동안 정상적인 사용 패턴을 관찰하여 기준선을 수립합니다:

- 시간당 평균 툴 호출 횟수와 분포
- 자주 호출되는 툴 목록과 파라미터 범위
- 외부 API 요청 대상 도메인 목록
- 응답 길이와 처리 시간 분포

이 기준선에서 일정 임계값 이상 벗어나는 패턴을 이상으로 탐지합니다.

## 거버넌스 관련 질문

### Q: APTS Tier 3 인증과 이 저장소의 관계는?

이 저장소는 APTS Tier 3 인증을 주장하지 않습니다. APTS Tier 1~3 기준을 참조하여
방어 원칙과 거버넌스 구조를 문서화한 것입니다. 실제 APTS 인증은 별도의 공식 프로세스를
통해 이루어집니다.

### Q: 이 저장소의 내용은 법적 컴플라이언스 근거로 활용할 수 있나요?

아니오. 이 저장소는 교육·연구 목적의 비공식 문서이며, 법적 컴플라이언스 근거로
활용할 수 없습니다. 규제 준수를 위해서는 관련 규정의 원문과 공인된 전문가 자문을
사용하세요.
