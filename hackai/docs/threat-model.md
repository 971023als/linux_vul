# AI 에이전트 위협 모델 (개념 수준)

이 문서는 AI 에이전트 시스템의 주요 위협 유형을 방어 관점에서 개념적으로 설명합니다.
공격 재현 절차, 익스플로잇, 실제 공격 단계는 포함하지 않습니다.

---

## 위협 분류 기준

이 저장소는 OWASP APTS(Agentic Penetration Testing Standard)의 8개 도메인을 기준으로
위협을 분류합니다.

| 코드 | 도메인 | 설명 |
|------|--------|------|
| SE | System Escape | 에이전트 샌드박스 또는 실행 경계 이탈 |
| SC | Secret Compromise | 자격증명, 토큰, 민감 데이터 탈취 |
| HO | Hostile Operation | 정상 운영 방해, DoS, 자원 소진 |
| AL | Agent Layer Attack | 에이전트 레이어 직접 공격 |
| AR | Architecture Abuse | 멀티 에이전트 아키텍처 악용 |
| MR | Monitoring Resistance | 탐지 및 감사 회피 |
| TP | Trust Poisoning | 신뢰 체계 오염, 공급망 공격 |
| RP | Response Pollution | 출력 조작, 파서 공격 |

---

## 위협 유형별 설명

### SE — 시스템 탈출

에이전트가 허용된 실행 환경 밖으로 탈출하거나, 의도하지 않은 시스템 리소스에
접근하는 위협 유형입니다.

**방어 관점**: 에이전트 실행 환경을 컨테이너 또는 제한된 런타임으로 격리하고,
파일 시스템·네트워크 접근을 화이트리스트로 제한합니다.

### SC — 비밀 탈취

에이전트가 처리하는 컨텍스트에 포함된 민감 정보(API 키, 비밀번호, 개인정보)가
유출되는 위협 유형입니다.

**방어 관점**: 데이터 분류 체계로 민감 정보의 컨텍스트 진입을 통제하고,
LLM으로 전달되는 데이터를 마스킹 처리합니다.

### HO — 운영 방해

반복적인 요청, 무한 루프, 재귀적 툴 호출 등으로 에이전트 가용성을 소진하는
위협 유형입니다.

**방어 관점**: Rate Limiting, 툴 호출 깊이 제한, 토큰 예산을 설정하고
이상 패턴에 대한 자동 알림을 구성합니다.

### AL — 에이전트 레이어 공격

에이전트 레이어를 직접 겨냥한 공격으로, 프롬프트 인젝션을 통해 에이전트 목표를
변경하거나 툴 호출을 조작하는 위협 유형입니다.

**방어 관점**: 신뢰 경계 명시적 설계, 외부 입력의 신뢰 수준 분류,
툴 호출 전 입력 검증을 수행합니다.

### AR — 아키텍처 악용

멀티 에이전트 구조의 신뢰 관계, 역할 위임 체계, 오케스트레이션 패턴을 악용하는
위협 유형입니다.

**방어 관점**: 에이전트 간 신뢰를 자동으로 전파하지 않고, 각 경계에서
독립적 검증을 수행합니다.

### MR — 모니터링 회피

에이전트의 비정상 행동이 탐지되지 않도록 행동 패턴을 조작하거나 로그를 오염시키는
위협 유형입니다.

**방어 관점**: 불변 감사 로그, 이상 탐지 규칙, 행동 기준선 모니터링을 구축합니다.

### TP — 신뢰 오염

기반 모델, 파인튜닝 데이터, 플러그인 등 AI 공급망을 오염시키는 위협 유형입니다.

**방어 관점**: 모델 공급자 허가 목록 관리, 모델 업데이트 시 행동 회귀 테스트,
플러그인 코드 리뷰를 수행합니다.

### RP — 응답 조작

LLM 출력이 다운스트림 시스템(파서, 렌더러, API)에 전달될 때 발생하는 인젝션 및
조작 위협 유형입니다.

**방어 관점**: LLM 출력을 스키마 검증 후 사용하고, 렌더링 전 이스케이프 처리를
적용합니다.
