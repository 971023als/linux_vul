# 방어 플레이북 (개념 수준)

이 문서는 AI 에이전트 시스템 설계 및 운영에서 적용할 수 있는
방어 전략을 개념적으로 정리합니다. 실제 공격 코드나 재현 절차는 포함하지 않습니다.

---

## 설계 단계 체크리스트

### 신뢰 모델 설계

- [ ] 에이전트가 처리하는 입력 채널을 모두 식별했는가?
- [ ] 각 채널의 신뢰 수준(높음/중간/낮음)을 문서화했는가?
- [ ] 낮은 신뢰 입력이 높은 신뢰 컨텍스트를 오염하지 못하도록 격리했는가?
- [ ] 외부 데이터(웹, 파일, DB)를 낮은 신뢰 등급으로 처리하는가?

### 권한 설계

- [ ] 에이전트에게 필요한 최소 권한만 부여했는가?
- [ ] 툴 목록이 화이트리스트로 명시되어 있는가?
- [ ] IAM 역할이 PoLP 원칙으로 설계되었는가?
- [ ] 미사용 권한 정기 감사 일정이 있는가?

### Human-in-the-Loop 설계

- [ ] 고위험 행동 목록을 사전에 정의했는가?
- [ ] 정의된 고위험 행동은 사람 승인을 거치도록 설계했는가?
- [ ] 승인 없이 실행된 경우 알림이 발생하는가?

---

## 구현 단계 체크리스트

### 입력 처리

- [ ] 외부 데이터를 컨텍스트에 적재하기 전 검증 단계가 있는가?
- [ ] SQL, 시스템 명령 등을 파라미터 바인딩으로 처리하는가?
- [ ] LLM 출력을 스키마 검증 후 사용하는가?

### 로깅 및 가시성

- [ ] 툴 호출 입출력이 구조화된 로그로 기록되는가?
- [ ] 로그가 불변 스토리지에 저장되는가?
- [ ] 에이전트 간 메시지가 기록되는가?

### 격리

- [ ] 에이전트 코드 실행이 격리된 환경에서 이루어지는가?
- [ ] 파일 시스템 접근이 허용된 경로로 제한되는가?
- [ ] 네트워크 접근이 화이트리스트로 제한되는가?

---

## 운영 단계 체크리스트

### 모니터링

- [ ] 에이전트 행동 기준선(baseline)이 정의되어 있는가?
- [ ] 이상 패턴에 대한 자동 알림이 설정되어 있는가?
- [ ] 정기적인 에이전트 행동 감사를 수행하는가?

### 공급망 관리

- [ ] 사용 중인 모델의 공급자가 검증되었는가?
- [ ] 모델 업데이트 시 행동 회귀 테스트를 수행하는가?
- [ ] 플러그인·라이브러리 취약점을 정기적으로 확인하는가?
