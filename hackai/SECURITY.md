# 보안 정책

## 이 저장소의 보안 철학

이 저장소는 **방어형 보안 연구 문서** 저장소입니다.  
공격 코드, 익스플로잇, 취약점 재현 절차는 포함하지 않으며, 이에 관한 신고는 별도 채널로 처리합니다.

---

## 지원 버전

| 버전 | 지원 여부 |
|------|-----------|
| 0.1.x | 지원 중 |
| < 0.1.0 | 미지원 |

---

## 취약점 신고 (Responsible Disclosure)

이 저장소의 코드 또는 문서에서 보안 취약점을 발견하셨다면, 아래 절차를 따라 주세요.

### 신고 채널

- **GitHub Issues 사용 금지**: 취약점 상세 내용을 공개 Issue로 등록하지 마세요.
- **Private Disclosure 우선**: 저장소 관리자에게 직접 비공개 연락을 주세요.
- **GitHub Security Advisory**: 이 저장소의 Security 탭 → "Report a vulnerability" 기능을 활용해 주세요.

### 신고 시 포함 내용

1. 취약점 유형 (예: XSS, 의존성 취약점, 잘못된 보안 설명)
2. 영향 범위 (어떤 파일, 어떤 기능에 해당하는지)
3. 재현 가능 여부 및 재현 환경
4. 제안하는 완화 방법 (선택 사항)

---

## 처리 절차

1. **접수 확인**: 신고 접수 후 48시간 이내 확인 응답
2. **영향 평가**: 관리자가 보안 영향도를 평가
3. **수정 및 릴리즈**: 수정 사항 반영 및 버전 배포
4. **공개 고지**: 수정 완료 후 적절한 시점에 공개 안내 (신고자 동의 하에 기여 기록 가능)

## 이 저장소 콘텐츠의 오용 신고

이 저장소의 콘텐츠가 비인가 시스템 공격 등 악의적 목적으로 사용되는 것을 발견한 경우:

- 해당 플랫폼(GitHub 등)의 신고 기능을 활용하세요.
- 이 저장소의 목적에 반하는 파생 저작물은 [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) 및 [LICENSE.md](LICENSE.md)에 따라 허용되지 않습니다.

## 의존성 보안

이 저장소는 Next.js 14 기반 정적 문서 사이트입니다.  
`npm audit`를 정기적으로 실행하여 의존성 취약점을 확인하세요.

```bash
npm audit
npm audit fix
```

알려진 고위험 취약점이 발견된 경우 Issue를 통해 알려 주시면 신속하게 처리합니다.
